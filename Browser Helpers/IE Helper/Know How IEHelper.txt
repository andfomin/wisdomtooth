You may try to debug by setting up the Command in the project properties to the browser executable. But the chrome shell starts initially without loading addons, it then starts a tab in another process.
On Windows Vista, the Internet Explorer Protected Mode feature launches a separate process and exits, making it a little harder to debug. You can easily turn off Protected Mode for the current session in two ways: launch the browser from a administrative process (such as Visual Studio), or create a local HTML file and specify it as a command line parameter to Internet Explorer.
---------------------------------------------
Our BHO will be istantiated gazillion times. A single error message can flood the event log. 
Thus do not report anything to user in the release build. Better die silently than interfere with the user's browsing experience. 
In the debug build report excessively. Write log messages to a file on disk. 
Since the logger is called from BHO in Protected Mode, it is permitted to write to folder FOLDERID_LocalAppDataLow only.
